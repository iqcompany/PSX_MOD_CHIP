# PSX MOD CHIP — AIによるHEXリバースエンジニアリングと移植

PlayStation 1のモッドチップのHEXファイルを、AIを使ってリバースエンジニアリングし、
別のマイコンに移植するまでの過程を記録したリポジトリです。

この内容は [note記事](https://note.com/iq_company/n/nb6c51826ef08) の素材として作成しました。

## やったこと

**手元にあった情報はこれだけ:**

- `psxmod.hex` — PIC12C508用のHEXファイル
- チップがPIC12C508であるという事実

ここからAI (Claude) との対話だけで以下を行いました:

1. **HEXファイルの逆アセンブル** — PIC12C508の12ビット命令セットをWebから調査し、HEXをデコードして `psx.asm` を生成
2. **コードの解析** — 各サブルーチンの役割、信号プロトコル、タイミングを解読し `psx.md` にレポート作成
3. **処理フローの可視化** — PlantUMLのアクティビティ図 `psx.pu` を作成
4. **別チップへの移植** — 6ピンの米粒PIC (PIC10F202) 向けに `psx_10F202.asm` を作成
5. **ハンドアセンブル** — アセンブラを使わず手動で機械語に変換し `psx_10F202.hex` を生成

## モッドチップの概要

PlayStation 1は起動時にディスクのウォブルグルーブから地域コード文字列を読み取り、
正規品かどうかを判定しています。このモッドチップは、その信号を模倣して
3地域のコード (SCEI / SCEA / SCEE) をすべて送信することで、
リージョンチェックとコピープロテクトを回避します。

```
送信データ: "SCEI" (日本) → "SCEA" (北米) → "SCEE" (欧州) → 繰り返し
プロトコル: 250bps, LSBファースト, 反転データ, 1スタート+2ストップビット
```

## PIC12C508 → PIC10F202 移植の変更点

| 項目 | PIC12C508 (8pin DIP) | PIC10F202 (6pin SOT-23) |
|------|----------------------|-------------------------|
| パッケージ | 8ピン | 6ピン (米粒サイズ) |
| メモリ | OTP (一回書込) | Flash (書換可能) |
| 発振器 | 外部クリスタル | 内蔵4MHz RC |
| 外付け部品 | クリスタル+コンデンサ×2 | なし |
| RAM開始 | 0x07 | 0x08 |
| 4.45MHz補正 | あり | 削除 (内蔵4MHz固定) |

```
PIC10F202:        PIC12C508:
  ┌──────┐          ┌──────┐
 1│VDD  6│VSS      1│VDD  8│VSS
 2│GP3  5│GP0      2│GP5  7│GP0
 3│GP1  4│GP2      3│GP4  6│GP1 ← DATA
  └──────┘         4│GP3  5│GP2 ← GATE
   DATA↑  ↑GATE     └──────┘
```
